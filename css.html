Tema Claro

CSS: Estilo e Apresentação de Páginas

CSS (Cascading Style Sheets) é a linguagem usada para definir aparência visual, cores, fontes, espaçamentos e layout de elementos HTML.

Função do CSS

  • Separar conteúdo (HTML) de apresentação (CSS).
  • Controlar layout, cores, tipografia e responsividade.
  • Permitir consistência visual em múltiplas páginas.

Exemplo Básico


body {
    font-family: Arial, sans-serif;
    background-color: #f5f5f5;
    color: #333;
}
h1 { color: #007acc; }

Vantagens do CSS

  • Manutenção simplificada: alterar um arquivo CSS altera todo o site.
  • Controle completo do layout e responsividade.
  • Melhor desempenho em comparação a estilos inline.

História e Evolução do CSS

CSS surgiu em 1996, criado por Håkon Wium Lie, para separar conteúdo de apresentação e facilitar manutenção de sites.

CSS1, CSS2 e CSS3

Resumo da evolução das versões com novas funcionalidades e modularização.

  • CSS1: estilos básicos (cores, fontes, espaçamentos).
  • CSS2: posicionamento, z-index e media types.
  • CSS3: flexbox, grid, animações e variáveis.

Sintaxe Básica do CSS

Composta por seletores e declarações com propriedades e valores.


p {
    color: #333;
    font-size: 16px;
    line-height: 1.5;
}

Seletores e Hierarquia

Define quais elementos HTML receberão determinado estilo.


a:hover { color: red; }
p::first-letter { font-size: 200%; }

Box Model e Layout

Todo elemento é uma caixa com conteúdo, padding, border e margin.


div {
    width: 200px;
    padding: 10px;
    border: 2px solid #000;
    margin: 20px;
}

Layout Avançado: Flexbox, Grid e Responsividade

Ferramentas modernas para layouts flexíveis e adaptáveis.


.container { display: flex; }
.grid-container { display: grid; }
@media (max-width: 768px) { .grid-container { grid-template-columns: 1fr; } }

Animações e Transições

Permitem criar efeitos visuais suaves sem JavaScript.


button { transition: background-color 0.3s; }
@keyframes exemplo { 0% { opacity: 0; } 100% { opacity: 1; } }

Boas Práticas de CSS

Organização, nomenclatura e performance para código limpo e sustentável.

  • Evite !important e seletores profundos.
  • Prefira classes reutilizáveis e variáveis CSS.
  • Comente blocos de layout e responsividade.

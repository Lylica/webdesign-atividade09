Tema Claro

CSS: Estilo e Apresentação de Páginas

CSS (Cascading Style Sheets) é a linguagem usada para definir aparência visual, cores, fontes, espaçamentos e layout de elementos HTML.

Função do CSS

  • Separar conteúdo (HTML) de apresentação (CSS).
  • Controlar layout, cores, tipografia e responsividade.
  • Permitir consistência visual em múltiplas páginas.

Exemplo Básico


body {
    font-family: Arial, sans-serif;
    background-color: #f5f5f5;
    color: #333;
}

h1 {
    color: #007acc;
}

Vantagens do CSS

  • Manutenção simplificada: alterar um arquivo CSS altera todo o site.
  • Controle completo do layout e responsividade.
  • Melhor desempenho em comparação a estilos inline.

História e Evolução do CSS

CSS surgiu em 1996, criado por Håkon Wium Lie, para separar conteúdo de apresentação e facilitar manutenção de sites.

CSS1 (1996)

Definia estilos básicos: cores, fontes, alinhamentos e espaçamentos.

CSS2 (1998)

Adicionou posicionamento, z-index, media types e pseudo-classes.

CSS3 (2000 em diante)

Modularizado em módulos, trouxe: flexbox, grid, transições, animações, sombras, gradientes, media queries e variáveis.

Impacto e Padrões

  • W3C: define especificações oficiais.
  • WHATWG: mantém interoperabilidade e boas práticas.
  • Navegadores: implementam CSS parcialmente; módulos podem ter suporte variável.

Sintaxe Básica do CSS

Um conjunto de regras CSS é composto por seletores e declarações. Cada declaração define propriedades e valores.

Estrutura


seletor {
    propriedade: valor;
    propriedade: valor;
}

Exemplo


p {
    color: #333;
    font-size: 16px;
    line-height: 1.5;
}

Formas de Inserção

  • Inline: <p style="color:red;">Texto</p>
  • Internal: dentro de <style></style> no <head>
  • External: arquivo style.css vinculado com <link>

Seletores e Hierarquia

Seletores identificam elementos HTML aos quais serão aplicados estilos.

Tipos de Seletores

  • Universal: * seleciona todos os elementos.
  • Elemento: p, h1
  • Classe: .menu
  • ID: #principal
  • Descendente: div p
  • Filho: div > p
  • Pseudo-classes: :hover, :focus
  • Pseudo-elementos: ::before, ::after

Exemplo de Pseudo-classes


a:hover {
    color: red;
    text-decoration: underline;
}

Exemplo de Pseudo-elementos


p::first-letter {
    font-size: 200%;
    color: #007acc;
}

Box Model e Layout

Todo elemento é uma caixa retangular composta por conteúdo, padding, border e margin.

Estrutura do Box Model

  • Content: conteúdo do elemento.
  • Padding: espaço interno entre conteúdo e borda.
  • Border: borda do elemento.
  • Margin: espaço externo entre elementos.

Exemplo Prático


div {
    width: 200px;
    padding: 10px;
    border: 2px solid #000;
    margin: 20px;
}

Propriedades de Layout

  • Display: block, inline, inline-block, flex, grid.
  • Position: static, relative, absolute, fixed, sticky.
  • Float e Clear: posicionamento tradicional de colunas.

Layout Avançado: Flexbox, Grid e Responsividade

CSS moderno permite criar layouts complexos e responsivos sem hacks.

Flexbox


.container {
    display: flex;
    justify-content: space-between;
    align-items: center;
}

.item {
    flex: 1;
    margin: 5px;
}

Grid


.grid-container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    gap: 10px;
}

Responsividade


@media (max-width: 768px) {
    .grid-container {
        grid-template-columns: 1fr;
    }
}

Animações e Transições

CSS permite animar propriedades e criar efeitos visuais suaves sem JavaScript.

Transições


button {
    background-color: blue;
    transition: background-color 0.3s ease;
}

button:hover {
    background-color: red;
}

Animações


@keyframes exemplo {
    0% { transform: translateX(0); }
    50% { transform: translateX(50px); }
    100% { transform: translateX(0); }
}

div {
    animation: exemplo 2s infinite;
}

Boas Práticas de CSS

Diretrizes para escrever CSS limpo, eficiente e sustentável.

1. Organização

  • Use indentação consistente.
  • Separe regras por seções: reset, tipografia, layout, componentes, responsividade.
  • Evite !important desnecessários.

2. Nomenclatura

  • Use nomes significativos para classes e IDs.
  • Considere metodologias como BEM para maior clareza.

3. Reutilização

  • Evite duplicação de regras.
  • Use variáveis CSS e custom properties.
  • Crie classes utilitárias para estilos repetidos.

4. Performance

  • Minimize arquivos CSS e combine quando possível.
  • Evite seletores muito complexos ou profundos.
  • Otimize imagens, fontes e ícones.

5. Acessibilidade e SEO

  • Considere contraste de cores e legibilidade.
  • Use animações e transições de forma moderada.
  • Prefira cores, fontes e layouts consistentes para melhorar UX.

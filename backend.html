Introdução ao Backend

Backend refere-se à parte do sistema que roda no servidor, responsável por processar dados, controlar lógica de negócio e gerenciar banco de dados.

Principais Responsabilidades

  • Processamento de dados e lógica do sistema.
  • Gerenciamento de bancos de dados e armazenamento.
  • Autenticação e segurança de usuários.
  • Comunicação via APIs com frontend.
  • Monitoramento e logging de aplicações.

Tipos de Backend

  • Monolítico: aplicação única que controla tudo.
  • Microserviços: cada serviço é independente e comunica via APIs.
  • Serverless: funções acionadas sob demanda, sem servidor persistente.

Node.js

Node.js é um runtime JavaScript para servidor, baseado em V8, permitindo criar aplicações rápidas e escaláveis.

Vantagens do Node.js

  • Single-threaded com event loop eficiente.
  • Grande ecossistema de pacotes (npm).
  • Mesma linguagem no front e back (JS).
  • Alta performance para aplicações I/O intensivas.

Exemplo de Servidor


// Servidor básico Node.js
const http = require('http');
const port = 3000;

const server = http.createServer((req, res) => {
    res.writeHead(200, {'Content-Type': 'text/plain'});
    res.end('Olá, mundo!\n');
});

server.listen(port, () => console.log(`Servidor rodando em http://localhost:${port}`));

Express.js

Framework minimalista para Node.js que facilita rotas, middlewares e APIs.

Roteamento


// Exemplo de rota
const express = require('express');
const app = express();
const port = 3000;

app.get('/', (req, res) => res.send('Olá, Express!'));
app.listen(port, () => console.log(`Servidor rodando em http://localhost:${port}`));

Parâmetros e Query


app.get('/usuario/:id', (req, res) => {
    res.send(`Usuário: ${req.params.id}`);
});

app.get('/busca', (req, res) => {
    res.send(`Query: ${req.query.q}`);
});

Bancos de Dados

O backend utiliza bancos de dados para armazenar e consultar informações de forma organizada.

Tipos de Banco

  • Relacional (SQL): MySQL, PostgreSQL – tabelas, joins, ACID.
  • NoSQL: MongoDB, Firebase – documentos, chave-valor, flexibilidade.

Exemplo com MongoDB (Mongoose)


const mongoose = require('mongoose');
mongoose.connect('mongodb://localhost:27017/teste');

const usuarioSchema = new mongoose.Schema({
    nome: String,
    idade: Number
});

const Usuario = mongoose.model('Usuario', usuarioSchema);
Usuario.create({nome: 'Ana', idade: 25});

APIs REST

APIs permitem que sistemas distintos se comuniquem de forma padronizada.

Princípios REST

  • HTTP Methods: GET, POST, PUT, DELETE
  • URLs claras e recursos bem definidos
  • Stateless: cada requisição independente
  • Uso de JSON para dados

Exemplo de API


app.get('/usuarios', (req, res) => {
    res.json([{id: 1, nome: 'Ana'}, {id: 2, nome: 'Pedro'}]);
});

Autenticação e Segurança

Garantir acesso seguro e proteger dados do usuário é fundamental.

JWT (JSON Web Token)


const jwt = require('jsonwebtoken');
const token = jwt.sign({id: 1}, 'segredo', {expiresIn: '1h'});
jwt.verify(token, 'segredo', (err, decoded) => console.log(decoded));

Proteção de Rotas


function auth(req, res, next) {
    const token = req.headers.token;
    if(token === '12345') next();
    else res.status(401).send('Não autorizado');
}
app.get('/dashboard', auth, (req, res) => res.send('Bem-vindo!'));

Middlewares

Middlewares são funções que interceptam requisições antes de chegarem às rotas.


// Middleware de logging
app.use((req, res, next) => {
    console.log(`${req.method} ${req.url}`);
    next();
});

// Middleware de tratamento de erros
app.use((err, req, res, next) => {
    console.error(err.stack);
    res.status(500).send('Erro interno');
});

Boas Práticas de Backend

  • Modularize código (rotas, controllers, services).
  • Use variáveis de ambiente para dados sensíveis.
  • Valide e sanitize entradas do usuário.
  • Trate erros e registre logs.
  • Implemente autenticação e HTTPS.
  • Documente APIs com Swagger ou similar.
  • Testes automatizados: unitários e integração.
